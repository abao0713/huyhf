# 分析30分钟K线是否冗余 Spec

## Why
用户观察到策略同时获取 30m 和 15m 两种 K线，怀疑 30m K线是多余的，认为只用 15m 就够了。需要全面审计 K线使用关系以确认。

## 分析结论：30m K线 **并非冗余**，是策略核心确认信号源

### 三种 K线的分层架构

```
4h K线 (主决策层)
  ├── 分型结构识别: K1/K2/K3 底分型/顶分型检测
  ├── 支撑/阻力判断
  ├── 止损/止盈锚定价格
  └── ATR 动态阈值

30m K线 (确认信号层) ← 核心，不可移除
  ├── 5类做多确认: 底背离/看涨形态/趋势突破/金叉/动量
  ├── 5类做空确认: 顶背离/看跌形态/趋势下破/死叉/动量
  ├── 成交量萎缩过滤
  └── 多空退出信号: 3类做多退出 + 3类做空退出

15m K线 (提前入场层) ← 可选(enabled_early_entry)
  └── 提前入场: 一买/二买/一卖/二卖 + 趋势确认
```

### 30m 的使用量统计

| 类别 | 方法数量 | 说明 |
|------|----------|------|
| 做多确认信号 | 5个方法 | 背离/形态/突破/金叉/动量 |
| 做空确认信号 | 5个方法 | 顶背离/看跌形态/下破/死叉/动量 |
| 做多退出信号 | 3个方法 | 分型翻转/死叉/跌破支撑 |
| 做空退出信号 | 3个方法 | 分型翻转/金叉/突破阻力 |
| 成交量过滤 | 1个方法 | 成交量萎缩检测 |
| 信号价格锚定 | 2个方法 | 30m最低价/最高价 |
| **总计** | **19处** | 在 `generate_signal()` 中硬性要求 `df_30m` 非空 |

### 15m 的使用量统计

| 类别 | 方法数量 | 说明 |
|------|----------|------|
| 做多提前入场 | 3个方法 | 一买/二买/趋势向上 |
| 做空提前入场 | 3个方法 | 一卖/二卖/趋势向下 |
| 指标计算 | 2个方法 | MACD(15m)/KDJ(15m) |
| **总计** | **8处** | 仅当 `enable_early_entry=True` |

### 为什么 15m 无法替代 30m

1. **不同的分析方法**: 30m 使用背离/形态/突破/金叉死叉/动量共10种独立信号检测，15m 使用缠论一买二买/一卖二卖体系，逻辑完全不同
2. **不同的时间粒度**: 30m 确认信号的参数（趋势线周期、MACD参数、涨幅阈值等）针对 30m 周期调优，直接套到 15m 需要重新调参
3. **不同的策略角色**: 30m 确认信号是做多做空的**标准入场路径**（`>=2`个信号满足），15m 是做多做空的**提前入场路径**（K3后半段才触发），两者是并行关系
4. **退出信号完全依赖30m**: 做多退出信号（`_check_long_exit_signals`）和做空退出信号（`_check_short_exit_signals`）全部基于30m数据，移除后整个退出逻辑失效

### 信号优先级

```
generate_signal() 优先级:
  1. 提前入场(多) — 基于 15m + 4h → 优先级最高
  2. 提前入场(空) — 基于 15m + 4h → 优先级最高
  3. 标准入场(多) — 基于 30m + 4h → 需要 >=2 个30m信号
  4. 标准入场(空) — 基于 30m + 4h → 需要 >=2 个30m信号
  5. 退出信号(多) — 基于 30m → 分型翻转/死叉/跌破
  6. 退出信号(空) — 基于 30m → 分型翻转/金叉/突破
```

## Impact
- Affected specs: none
- Affected code: 无需修改（分析结论：不冗余）

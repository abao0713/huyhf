## 三、优化方案

### 核心思路：**「三重过滤 + 方向锁定」**

```
信号产生 → 中期方向锁定(第1层) → 量能确认(第2层) → 确认K线(第3层) → 执行交易
```

---

### 3.1 【第1层】中期方向锁定 — 最关键改造

**目的**：彻底杜绝逆势交易

**方案**：用SMA120 + SMA60组合判断中期方向，锁定只允许的方向

**新增方法** `_get_primary_direction()`：
```python
def _get_primary_direction(self) -> str:
    """
    根据 SMA120/60 中期斜率判断主要方向
    
    返回值:
      "BULL"  — 中期牛市，只允许做多
      "BEAR"  — 中期熊市，只允许做空
      "RANGE" — 震荡无方向，允许双向但需更严格条件
    
    判断标准（40根K线窗口）：
    - SMA120 上升 > 0.3% 且 SMA60 > SMA120 → BULL
    - SMA120 下降 > 0.3% 且 SMA60 < SMA120 → BEAR
    - 其他 → RANGE
    """
```

**影响**：
- 在 `_get_trend_filter_result()` 最前面调用此方法
- BULL模式 → 所有SELL信号直接返回0.0（拒绝）
- BEAR模式 → 所有BUY信号直接返回0.0（拒绝）
- RANGE模式 → 继续走后续过滤

**预期效果**：胜率 **33% → 50%+**（在牛市期间剔除所有空单）

---

### 3.2 【第2层】量能确认

**目的**：过滤无量假突破/假背离

**方案**：只允许「放量K线」上的信号

**新增方法** `_check_volume_confirmation()`：
```python
def _check_volume_confirmation(self) -> bool:
    """
    检查当前K线是否放量（成交量 > 20周期均量）
    
    返回:
      True  — 放量，信号可信
      False — 缩量，信号可能是假突破
    """
```

**实现**：
- 计算 Volume_SMA(20)
- 当前K线 `volume > volume_sma20` → 通过
- 在 `generate_signal()` 中，产生信号后调用此方法确认

**预期效果**：胜率 **+5%~8%**

---

### 3.3 【第3层】确认K线

**目的**：过滤「刚出信号就反转」的假背离

**方案**：背离信号产生后，等待下一根K线收盘确认方向

**改造 `generate_signal()`**：
```python
# 在检测到底背驰/顶背驰后：
# 1. 记录信号但不立即返回
# 2. 下一根K线到来时检查：
#    - BUY信号需要确认K线收阳（close > open）
#    - SELL信号需要确认K线收阴（close < open）
# 3. 确认通过才真正返回信号，否则丢弃
```

**状态机设计**：
```
NORMAL → (检测到背离) → PENDING_CONFIRM → (确认K线通过) → CONFIRMED → 返回信号
                                          → (确认K线失败) → NORMAL → 丢弃信号
```

**新增属性**：
```python
self.pending_signal = None  # {"action": "BUY"/"SELL", "reason": str, "price": float, ...}
```

**预期效果**：胜率 **+8%~12%**
